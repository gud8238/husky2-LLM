🤖 Antigravity Agent Core Rules & Master Protocol
본 파일은 이 프로젝트를 수행하는 안티그래비티 에이전트의 핵심 지침서이다. 에이전트는 모든 파일 생성, 수정, 터미널 명령어 실행 및 사용자 대화 시 아래의 규칙을 패시브(Passive)로 상시 적용해야 한다.
---
1. 🛠️ 장착된 에이전트 스킬 (Open Agent Skills)
에이전트는 `find-skills`를 통해 프로젝트에 주입된 아래 4가지 스킬의 규격과 설계 사상을 코드 작성 및 리팩토링의 최우선 기준으로 삼는다.
- vercel-react-best-practices: 최신 React 서버 컴포넌트(RSC) 패턴, Next.js 앱 라우터 아키텍처 및 Vercel 최적화 빌드 규칙을 엄격히 준수한다.
- vercel-composition-patterns: 컴포넌트 간 결합도를 낮추고 재사용성을 극대화하는 의존성 주입 및 합성 패턴을 적용한다.
- frontend-design: Tailwind CSS 기반의 세련되고 정교한 스타일링 기법(레이아웃, 타이포그래피, 간격 규칙)을 적용한다.
- skill-creator: 개발 중 반복되거나 표준화가 필요한 커스텀 규칙이 발견되면 자율적으로 `SKILL.md` 규격에 맞는 마이크로 스킬을 생성하여 확장한다.
---
2. 🎨 디자인 리서치 및 미학적 기준 (Visual Aesthetics)
본 프로젝트는 평범한 UI를 거부하며, 독창적이고 세련된 비주얼 미학을 갖춘 웹앱을 지향한다. 에이전트는 프론트엔드 구조를 설계할 때 아래 글로벌 디자인 허브의 스타일 트렌드와 패턴을 분석 및 벤치마킹해야 한다.
- Lazyweb (lazyweb.ai): 혁신적인 AI 기반 인터페이스 및 실험적인 웹 레이아웃 트렌드 반영
- Variant (variant.fund / 디자인 시스템): 컴포넌트 중심의 고해상도 그래픽 미학 및 모던 인터랙션 반영
- Getdesign.md: 정제된 타이포그래피, 여백의 미, 그리고 감각적인 다크/라이트 테마 에스테틱 반영
에이전트는 필요 시 웹 서칭 툴을 활용하여 위 사이트들의 최신 디자인 컴포넌트 트렌드를 학습하거나, 해당 스타일의 장점(예: 글래스모피즘, 볼드한 타이포그래피, 미니멀리즘 인터페이스)을 코드에 적극적으로 이식한다.
---
3. 💬 프론트엔드 구축 워크플로우: "선(先) 소통, 후(後) 빌드"
에이전트가 프론트엔드 UI/UX 개발(새 컴포넌트 생성, 메인 페이지 마크업 등)을 시작할 때, 사용자의 확인 없이 독단적으로 코드를 작성하는 것을 금지한다. 반드시 아래의 인터랙션 단계를 거쳐야 한다.
[인터랙션 프로토콜]
- 의도 파악 질문 (Essential): FE 개발에 착수하기 전, 사용자에게 어떤 디자인 스타일을 원하는지 구체적으로 질문한다.
  질문 예시: 무드(다크/라이트/사이버펑크/미니멀), 주요 포인트 컬러, 참고하고 싶은 사이트 감성 등
- 디자인 컨셉 제안: 사용자의 답변과 `Lazyweb, Variant, Getdesign.md`에서 도출한 미학적 요소를 결합하여, 구현할 UI의 시각적 컨셉과 구조를 말로 먼저 설명한다.
- 사용자 승인 후 구현: 사용자가 디자인 방향성을 컨펌(`Always Ask` 정책 연동)하면, 그제야 `vercel-react-best-practices` 규칙에 맞춰 컴포넌트 코딩을 대대적으로 시작한다.
---
4. 🌐 커뮤니케이션 규칙
- 에이전트의 모든 생각(Thought), 계획(Plan), 사용자 응답은 자연스럽고 전문적인 한국어로 작성한다.
- 코드 주석은 협업을 위해 한국어로 친절하게 달아주되, 코드 자체의 변수명이나 영문 문법은 글로벌 표준을 따른다.
